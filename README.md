# mkpull-pages

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 文件的集合与托管空间。

## 项目简介

`mkpull-pages` 旨在提供一个简洁、通用的 HTML 页面归档与发布方案。通过将多个独立的 HTML 文件集中存储于此仓库，用户可以方便地管理、版本控制并借助 GitHub Pages 功能实现静态页面的在线访问。

## 目录结构

```
.
├── README.md          # 本文件
├── index.html         # 可选：仓库首页或导航页
├── pages/             # 存放主要 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/`：存放所有独立 HTML 页面文件，每个文件可视为一个独立的归档单元。
- `assets/`：存放页面共享的样式、脚本或媒体资源，便于统一维护。

## 页面归档说明

- 每个 HTML 文件应保持自包含，推荐将样式和脚本内联或引用 `assets/` 下的公共资源。
- 文件名建议使用英文小写字母、数字与连字符，避免特殊字符。
- 若页面之间存在关联，可通过链接互相跳转，但每个页面均需独立可用。
- 归档页面不包含任何具体网站或商业推广内容，仅用于展示或功能演示。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建新的 HTML 文件，并按需更新 `assets/` 或 `index.html`。
- 修改已有页面：直接编辑对应文件，提交前请确保页面在本地可正常渲染。
- 删除页面：移除对应文件，并清理 `index.html` 中的相关链接（如有）。
- 建议每次提交附带简洁的提交信息，说明增删改的内容。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 下添加或编辑 HTML 文件。
3. 提交并推送至 GitHub。
4. 启用 GitHub Pages 功能（通常选择 `main` 分支的根目录），即可通过对应 URL 访问页面。
